변수 선언

함수 내부에서 'var 변수명;' 으로 선언하면 지역 변수가 됨
함수 내부에서 '변수명;'으로 선언하거나 함수 밖에서'변수명;' 혹은 'var 변수명;'으로 선언하면 전역 변수가 됨
전역 변수와 지역 변수가 동일한 이름일 경우 함수 내부에서 변수 명을 호출하면 지역 변수가 우선권을 가짐
전역 변수 사용시 'windows.변수 명'혹은 'this.변수 명'으로 표현하며 지역 변수와 구분
전역 변수는 해당window내에서 어디든 사용할 수 있으며, 지역 변수는 해당 함수 내에서만 사용 가능

자료형

자바스크립트에서는 자료형 별로 변수 타입이 지정되지 않고 리터럴에 의해 자료형 결정
선언 시 자료형을 지정하지 않기 때문에 변수 명만으로 데이터의 자료형을 확인할 수 없으므로 값의 자료형을 확인하기 위해 typeof() 연산자 사용

var와 let과 const

기본 ES5버전에서는 모든 변수를 var자료형으로 받아왔지만 데이터의 모호함이라는 문제를 개선하기 위해 최근에는 ES6버전의 let과 const 자료형을 많이 쓰는 추세 -> 가독성 및 유지보수 증가
- let : 일반적인 변수 선언(덮어쓰기 가능, 기본 var와 유사)
- const : 상수 변수 선언(덮어쓰기 불가능)

for in

for of

for in은 객체의 key 또는 배열의 index를 반복하는 반복문이다. 객체에서 사용하면 객체의 속성 이름(key)을 하나씩 가져오고, 배열에서 사용하면 배열의 값이 아니라 index 번호(0,1,2…)를 가져온다. 그래서 배열에서 for in을 사용할 경우 실제 값이 아니라 index가 나오기 때문에 값을 사용하려면 배열[index] 형태로 다시 접근해야 한다. for in은 특히 객체의 속성을 순회할 때 사용하는 것이 일반적이다.

for of는 반복 가능한(iterable) 데이터에서 실제 값(value)을 하나씩 가져오는 반복문이다. 배열이나 문자열처럼 반복 가능한 데이터에서 사용하면 index가 아니라 실제 값 자체가 바로 반복 변수에 들어온다. 그래서 배열을 반복할 때는 for of가 훨씬 직관적이고 많이 사용된다.

또 하나 중요한 차이는 사용 가능한 대상이다. for of는 iterable 객체에서만 사용할 수 있기 때문에 배열, 문자열, Map, Set 등에서는 사용할 수 있지만 일반 객체(object)는 iterable이 아니기 때문에 사용할 수 없다. 그래서 객체에 for of를 사용하면 “object is not iterable” 같은 에러가 발생한다. 반대로 for in은 객체의 key를 기준으로 동작하기 때문에 객체에서 정상적으로 사용할 수 있다.

정리하면 for in은 key나 index를 반복할 때 사용하는 반복문이고, for of는 실제 값(value)을 반복할 때 사용하는 반복문이다. 그래서 일반적으로 객체를 반복할 때는 for in을 사용하고, 배열을 반복할 때는 for of를 사용하는 경우가 가장 많다. 또한 배열에서 for in을 사용할 경우 값이 아니라 index가 나온다는 점과, 객체에서는 for of를 사용할 수 없다는 점을 특히 조심해야 한다.
