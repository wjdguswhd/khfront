변수와 자료형

변수 선언

함수 내부에서 'var 변수명;' 으로 선언하면 지역 변수가 됨
함수 내부에서 '변수명;'으로 선언하거나 함수 밖에서'변수명;' 혹은 'var 변수명;'으로 선언하면 전역 변수가 됨
전역 변수와 지역 변수가 동일한 이름일 경우 함수 내부에서 변수 명을 호출하면 지역 변수가 우선권을 가짐
전역 변수 사용시 'windows.변수 명'혹은 'this.변수 명'으로 표현하며 지역 변수와 구분
전역 변수는 해당window내에서 어디든 사용할 수 있으며, 지역 변수는 해당 함수 내에서만 사용 가능

자료형

자바스크립트에서는 자료형 별로 변수 타입이 지정되지 않고 리터럴에 의해 자료형 결정
선언 시 자료형을 지정하지 않기 때문에 변수 명만으로 데이터의 자료형을 확인할 수 없으므로 값의 자료형을 확인하기 위해 typeof() 연산자 사용

var와 let과 const

기본 ES5버전에서는 모든 변수를 var자료형으로 받아왔지만 데이터의 모호함이라는 문제를 개선하기 위해 최근에는 ES6버전의 let과 const 자료형을 많이 쓰는 추세 -> 가독성 및 유지보수 증가
- let : 일반적인 변수 선언(덮어쓰기 가능, 기본 var와 유사)
- const : 상수 변수 선언(덮어쓰기 불가능)
